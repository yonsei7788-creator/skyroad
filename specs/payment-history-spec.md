# 결제내역 페이지 명세

## 1. 페이지 경로 및 메뉴 위치

- **경로**: `/profile/payments`
- **프로필 메뉴 순서**: 컨설팅 내역 → **결제내역** → 목표 대학 수정 → 추천인 코드 → 내 정보 수정

---

## 2. 표시할 데이터 항목

orders + payments + plans 조인으로 구성. 결제 완료 건(payments.status = 'done')만 표시.

| 항목      | 소스                   | 표시 방식                     |
| --------- | ---------------------- | ----------------------------- |
| 결제일    | payments.approved_at   | YYYY.MM.DD HH:mm              |
| 플랜명    | plans.display_name     | 라이트 / 스탠다드 / 프리미엄  |
| 결제금액  | payments.amount        | ₩59,000 형식                  |
| 할인금액  | orders.discount_amount | 0이면 미표시                  |
| 결제수단  | payments.method        | 카드 / 가상계좌 / 간편결제 등 |
| 주문 상태 | orders.status          | 상태 배지 (아래 참고)         |
| 주문번호  | payments.toss_order_id | 하단 작은 글씨                |

---

## 3. 정렬/필터 규칙

- **기본 정렬**: payments.approved_at 내림차순 (최신 결제 먼저)
- **필터**: 별도 필터 없음 (결제 건수가 적어 불필요)
- **페이징**: 전체 목록 표시 (무한 스크롤/페이지네이션 없음)

---

## 4. 빈 상태 처리

결제 내역이 없을 때:

> "아직 결제 내역이 없어요."
> [리포트 구매하기] 버튼 → `/pricing` 또는 결제 페이지로 이동

---

## 5. 상태별 표시 텍스트/배지

orders.status 기준:

| status            | 배지 텍스트 | 색상   |
| ----------------- | ----------- | ------ |
| pending_payment   | 결제 대기   | 회색   |
| paid              | 결제 완료   | 파란색 |
| analyzing         | 분석 중     | 노란색 |
| analysis_complete | 분석 완료   | 초록색 |
| under_review      | 검토 중     | 노란색 |
| review_complete   | 검토 완료   | 초록색 |
| delivered         | 전달 완료   | 초록색 |

> payments.status = 'canceled'인 경우 "결제 취소" 배지(빨간색)로 표시, 별도 목록 분리 없이 함께 표시.

---

## 6. 모바일 대응

- 카드형 레이아웃 (테이블 미사용)
- 한 카드 내 구성: 상단 좌측 플랜명 + 우측 배지 / 중단 결제금액(강조) + 결제수단 / 하단 결제일 + 주문번호(작은 글씨)
- 할인금액 있는 경우 정가에 취소선 + 실결제금액 표시
