## 1. 개요

### 1.1 목적

학생의 비교과 활동(세특, 창체, 활동 키워드 등)을 **입학사정관 관점**에서 정밀 분석하여, 현재 비교과의 합격 경쟁력 수준과 구체적 개선 전략을 제공한다.

기존 리포트가 성적 중심의 정량 분석에 강점이 있다면, 이 섹션은 **"이 학생의 비교과가 입시에서 얼마나 설득력 있는가"**에 집중한다.

### 1.2 핵심 설계 결정: 고정 템플릿 방식

이 섹션은 **AI가 문장을 생성하지 않는다.** 비교과 수준(상/중/하)에 따라 **미리 작성된 고정 멘트**가 조건 분기로 출력되고, 변수 슬롯(`{전공 방향}`, `{키워드}` 등)에만 학생별 데이터가 삽입된다.

```
┌─────────────────────────────────────────────┐
│  입력 데이터 (기존 파이프라인 추출값)         │
│  - 비교과 수준: 상/중/하                     │
│  - 전공 방향, 키워드, 세특 수준, 연결성       │
└──────────────────┬──────────────────────────┘
                   ▼
┌─────────────────────────────────────────────┐
│  조건 분기 (레벨별 고정 멘트 선택)            │
│  if 상 → 상위권 멘트 세트                    │
│  if 중 → 중위권 멘트 세트                    │
│  if 하 → 하위권 멘트 세트                    │
└──────────────────┬──────────────────────────┘
                   ▼
┌─────────────────────────────────────────────┐
│  변수 치환                                   │
│  {전공 방향} → "경영"                        │
│  {키워드1} → "데이터 분석"                    │
│  {키워드 기반} → "데이터 분석, 마케팅 기획"    │
└──────────────────┬──────────────────────────┘
                   ▼
             최종 렌더링
```

### 1.3 위치

| 항목      | 내용                                              |
| --------- | ------------------------------------------------- |
| 섹션 ID   | `competitiveProfiling`                            |
| 섹션명    | 비교과 경쟁력 정밀 분석                           |
| 위치      | **섹션 16 (admissionStrategy) 다음**, Part 3 전략 |
| 대상 플랜 | Lite, Standard, Premium (전 플랜, 동일 구조)      |
| 예상 분량 | 2~3P                                              |

### 1.4 기존 섹션과의 차별점

| 기존 섹션               | 역할                         | 이 섹션과의 차이                                     |
| ----------------------- | ---------------------------- | ---------------------------------------------------- |
| competencyScore (#2)    | 4대 역량 정량 스코어 (300점) | 숫자 기반 평가 vs **입시 설득력 관점 평가**          |
| activityAnalysis (#9)   | 창체 활동 개별 분석          | 활동별 상세 분석 vs **전체 비교과의 경쟁 구조 분석** |
| weaknessAnalysis (#13)  | 부족한 부분 + 보완 전략      | 전반적 약점 vs **비교과 한정 탈락 구조 + 전환 전략** |
| admissionStrategy (#16) | 입시 전략 + 대학 추천        | 전형/대학 중심 vs **비교과 경쟁력 중심**             |

## 2. 입력 데이터

기존 파이프라인에서 이미 추출/가공된 데이터를 활용한다. **별도 사용자 입력 없음.**

### 2.1 필요 데이터 4개

| #   | 입력 항목            | 타입                 | 데이터 소스                                        | 산출 방식                             |
| --- | -------------------- | -------------------- | -------------------------------------------------- | ------------------------------------- |
| 1   | **비교과 수준**      | `상 / 중 / 하`       | `competencyScore`                                  | 진로역량 + 공동체역량 점수 기반 3분위 |
| 2   | **전공 방향**        | `string`             | `studentProfile`                                   | 희망학과 또는 AI 추출 전공 방향       |
| 3   | **핵심 활동 키워드** | `string[]` (2~3개)   | `activityAnalysis`, `studentProfile.characterTags` | AI가 세특/창체에서 추출한 핵심 키워드 |
| 4   | **연결성**           | `있음 / 보통 / 없음` | `activityAnalysis`, `storyAnalysis`                | 활동 간 주제 연결도 분석 결과         |

> **참고**: 원문 문서의 "세특 수준"은 "비교과 수준"과 동일한 값이므로 별도 항목으로 분리하지 않는다.

### 2.2 비교과 수준 산출 기준

| 수준        | 조건                           | 설명                               |
| ----------- | ------------------------------ | ---------------------------------- |
| 상 (상위권) | 진로+공동체 합산 상위 30% 이상 | 비교과 활동이 풍부하고 방향성 있음 |
| 중 (중위권) | 상위 30~70% 구간               | 활동은 있으나 차별성 부족          |
| 하 (하위권) | 하위 30% 미만                  | 평가 가능한 정보 부족              |

### 2.3 연결성 산출 기준

| 수준 | 조건                               | 설명                       |
| ---- | ---------------------------------- | -------------------------- |
| 있음 | 3개 이상 활동이 전공 방향으로 연결 | 스토리라인이 명확히 형성됨 |
| 보통 | 1~2개 활동이 전공과 연결           | 부분적 연결, 설명 필요     |
| 없음 | 전공 방향과 활동 간 연결 미약      | 스토리라인 부재            |

## 3. 고정 멘트 사전 (Template Dictionary)

모든 출력 문구는 아래 사전에서 레벨별로 선택된다. `{변수}`는 학생별 데이터로 치환.

### 3.0 핵심 요약

**현재 상태:**

| 수준 | 고정 멘트                                                                                          |
| ---- | -------------------------------------------------------------------------------------------------- |
| 상   | "합격권에는 진입했으나, 최종 선발을 결정짓는 차별성은 다소 부족한 구조"                            |
| 중   | "탈락 사유가 명확하게 없지만, 합격 이유도 부족한 구조. 사정관 입장에서 많이 고민할 수 있는 생기부" |
| 하   | "평가 가능한 정보가 부족하여 경쟁력 판단이 어려운 구조"                                            |

**합격 분기점** (공통):

> "{전공 방향} 전공에 대한 설득력"과 "활동 간 연결 구조 형성 여부"에서 결과가 결정될 가능성이 높습니다.

### 3.1 정량 분석

| 수준 | 종합 점수 범위 | 구간 라벨   |
| ---- | -------------- | ----------- |
| 상   | 70~85점        | 합격권      |
| 중   | 50~65점        | 경쟁 구간   |
| 하   | 30~50점        | 비경쟁 구간 |

**핵심 해석** (공통):

> 현재 평가는 단순한 우열이 아니라 "얼마나 명확하게 해석되는가"에 따라 갈리는 구간입니다.

### 3.2 현재 학생 특징

**표시 항목:**

- 핵심 활동: `{키워드1}`, `{키워드2}`, `{키워드3}`
- 비교과 수준: `{비교과 수준}`
- 연결성: `{연결성}`

**평가 결과:**

| 수준 | 고정 멘트                                     |
| ---- | --------------------------------------------- |
| 상   | "최종 선발 후보군까지 안정적으로 진입 가능"   |
| 중   | "1차 통과 후, 최종 단계에서 탈락 가능성 존재" |
| 하   | "초기 평가 단계에서 경쟁력 부족 판단 가능성"  |

### 3.3 실제 탈락 구조

**공통 핵심 문장** (전 레벨):

> "현재 탈락은 부족해서가 아니라, 평가 과정에서 '명확하게 설명되지 않기 때문'에 발생합니다."

**레벨별 추가:**

| 수준 | 고정 멘트                                    |
| ---- | -------------------------------------------- |
| 상   | "유사한 상위권 학생 간 비교에서 밀리는 구조" |
| 중   | "비슷한 학생들 사이에서 구분되지 않는 구조"  |
| 하   | "학생의 강점이 충분히 드러나지 않는 구조"    |

### 3.4 합격 전환 메커니즘

**핵심 조건 3가지** (공통, 변수 치환):

| #   | 항목             | 고정 멘트                                               |
| --- | ---------------- | ------------------------------------------------------- |
| 1   | 핵심 방향성 정의 | "이 학생은 {키워드 기반}으로 {전공 방향}을 준비한 학생" |
| 2   | 활동 재해석      | "{키워드 활동}을 중심으로 모든 경험 재정렬"             |
| 3   | 세특 구조 강화   | "탐구 → 결과 → 의미 → 진로 연결"                        |

**예상 변화** (공통 고정):

1. 이해도 상승
2. 평가 시간 단축
3. 기억에 남는 학생으로 전환
4. → 결과: 합격 가능성 상승

### 3.5 경쟁자 대비 핵심 차이

**차이 발생 영역** (공통 고정):

1. 방향성 명확도
2. 메시지 전달력
3. 구조 완성도

**핵심 차이** (공통 고정):

> "경쟁자는 한 문장으로 설명되지만, 현재 학생은 설명이 필요한 상태입니다."

### 3.6 입학사정관 한줄 평가

(공통, 변수 치환):

> "{전공 방향} 전공에 대한 관심과 준비는 확인되지만, 선발을 결정할 만큼의 명확한 설득력은 다소 부족하다."

### 3.7 실행 전략

**반드시 해야 할 3가지** (공통, 변수 치환):

| #   | 항목             | 고정 멘트                                               |
| --- | ---------------- | ------------------------------------------------------- |
| 1   | 핵심 캐릭터 정의 | "이 학생은 {키워드 기반}으로 {전공 방향}을 준비한 학생" |
| 2   | 활동 구조 재정리 | "핵심 활동 중심으로 강조"                               |
| 3   | 세특 구조 통일   | "탐구 흐름 기반 재구성"                                 |

### 3.8 최종 결론

| 수준 | 고정 멘트                                                     |
| ---- | ------------------------------------------------------------- |
| 상   | "합격 가능성은 높지만, 최종 선발을 위해서는 차별화가 필요"    |

## 4. 데이터 구조

### 4.1 저장 구조 (TypeScript)

AI가 생성하는 것이 아니므로, 백엔드에서 저장하는 데이터는 **렌더링에 필요한 입력값만** 저장한다.

```typescript
/** 비교과 수준 등급 */
type NonAcademicLevel = "상" | "중" | "하";

/** 활동 연결성 등급 */
type ActivityConnectivity = "있음" | "보통" | "없음";

/** 비교과 경쟁력 정밀 분석 - 저장 데이터 */
interface CompetitiveProfilingSection {
  /** 비교과 수준 (레벨 분기 키) */
  level: NonAcademicLevel;

  /** 전공 방향 (변수 치환용) */
  majorDirection: string;

  /** 핵심 활동 키워드 2~3개 (변수 치환용) */
  keywords: string[];

  /** 활동 연결성 */
  connectivity: ActivityConnectivity;

  /** 정량 점수 (0~100, 레벨 기반 산출) */
  score: number;
}
```

### 4.2 산출 로직

AI 프롬프트가 아닌 **코드 로직**으로 산출한다. postprocessor 또는 별도 태스크에서 처리.

```typescript
// 비교과 수준 산출
function calculateNonAcademicLevel(
  competencyScore: CompetencyScoreSection
): NonAcademicLevel {
  const careerScore = competencyScore.career; // 진로역량 점수
  const communityScore = competencyScore.community; // 공동체역량 점수
  const combined = careerScore + communityScore; // 200점 만점

  if (combined >= 140) return "상"; // 상위 30%
  if (combined >= 80) return "중"; // 중위
  return "하"; // 하위 30%
}

// 정량 점수 산출
function calculateProfilingScore(level: NonAcademicLevel): number {
  switch (level) {
    case "상":
      return randomInRange(70, 85);
    case "중":
      return randomInRange(50, 65);
    case "하":
      return randomInRange(30, 50);
  }
}
```

## 5. 프론트엔드 렌더링

### 5.1 핵심 구현 방식

**고정 멘트는 프론트엔드 상수로 관리한다.** DB에서 받는 것은 `level`, `majorDirection`, `keywords`, `subjectRecordLevel`, `connectivity`, `score` 6개 값뿐이며, 나머지는 프론트엔드에서 조건 분기 + 변수 치환으로 렌더링한다.

```typescript
// 프론트엔드 상수 예시
const TEMPLATES = {
  summary: {
    상: "합격권에는 진입했으나, 최종 선발을 결정짓는 차별성은 다소 부족한 구조",
    중: "탈락 사유가 명확하게 없지만, 합격 이유도 부족한 구조. 사정관 입장에서 많이 고민할 수 있는 생기부",
    하: "평가 가능한 정보가 부족하여 경쟁력 판단이 어려운 구조",
  },
  evaluationResult: {
    상: "최종 선발 후보군까지 안정적으로 진입 가능",
    중: "1차 통과 후, 최종 단계에서 탈락 가능성 존재",
    하: "초기 평가 단계에서 경쟁력 부족 판단 가능성",
  },
  rejectionCause: {
    상: "유사한 상위권 학생 간 비교에서 밀리는 구조",
    중: "비슷한 학생들 사이에서 구분되지 않는 구조",
    하: "학생의 강점이 충분히 드러나지 않는 구조",
  },
  finalConclusion: {
    상: "합격 가능성은 높지만, 최종 선발을 위해서는 차별화가 필요",
    중: "합격 가능성은 있으나, 현재 구조로는 선택받기 어려운 상태",
    하: "현재는 부족하지만, 방향 설정에 따라 충분히 개선 가능한 상태",
  },
  zone: {
    상: "합격권",
    중: "경쟁 구간",
    하: "비경쟁 구간",
  },
} as const;

// 변수 치환 함수
function fillTemplate(
  template: string,
  data: CompetitiveProfilingSection
): string {
  return template
    .replace("{전공 방향}", data.majorDirection)
    .replace("{키워드 기반}", data.keywords.join(", "))
    .replace("{키워드 활동}", data.keywords[0]);
}
```

### 5.2 섹션 레이아웃

```
[비교과 경쟁력 정밀 분석]
├── 0. 핵심 요약 (카드: 현재 상태 + 합격 분기점)
├── 1. 정량 분석 (게이지 0~100 + 구간 뱃지)
├── 2. 현재 학생 특징 (키워드 태그 + 세특/연결성 + 평가 결과)
├── 3. 실제 탈락 구조 (공통 문장 + 레벨별 원인 강조 박스)
├── 4. 합격 전환 메커니즘 (3단계 스텝 + 예상 변화)
├── 5. 경쟁자 대비 핵심 차이 (3개 영역 + 핵심 차이 인용)
├── 6. 입학사정관 한줄 평가 (인용 박스)
├── 7. 실행 전략 (체크리스트 3개)
└── 최종 결론 (레벨별 색상 배너)
```

## 6. 파이프라인 통합

### 6.1 구현 방식: postprocessor 추가

AI 태스크가 아니므로, **기존 postprocessor에서 다른 섹션 결과를 조합하여 산출**한다.

```
기존 태스크 완료 → postprocessor에서 competitiveProfiling 산출
  1. competencyScore에서 비교과 수준(상/중/하) 산출
  2. studentProfile에서 전공 방향, 키워드 추출
  3. activityAnalysis에서 연결성 산출
  4. 정량 점수 계산
  5. CompetitiveProfilingSection 객체 생성 → report content에 저장
```

### 6.2 의존 데이터

| 소스 섹션          | 추출 데이터                                           |
| ------------------ | ----------------------------------------------------- |
| `competencyScore`  | 진로역량 점수, 공동체역량 점수 → 비교과 수준 산출     |
| `studentProfile`   | `majorDirection`, `characterTags` → 전공 방향, 키워드 |
| `activityAnalysis` | 활동 연결도 → 연결성 산출                             |

### 6.3 플랜별 차이

전 플랜 동일 구조, 동일 출력.
